hexo + github page 建站中的坑

发表于 2017-08-03

本文主要讲述如何使用hexo生成静态网页并在github page上托管中碰到的坑。

过程

很多博客均有仔细介绍如何使用hexo+github建站的流程，博主主要参考barry的建站，其描写比较细致，不再一一列出，主要分为三步，安装hexo，部署github，解析域名。

Note:

  • 建议先阅读官方文档，以便熟悉指令
  • 配置git时，如果发现ssh到git失败，可能是git包没安装可以使用指令 npm install hexo-deployer-git --save , 还有可能ssh key没复制对。（如果你没联网就算了，当我没说前面的话）

hexo 生成静态文件时，几个常用的指令交代下：
hexo g
用于生成静态网址，这里发现generate的时候，会把theme里的image覆盖到public文件夹，因此主页修改个人头像建议把theme。的/image/avatar.gif替代成自己想要的头像。
hexo d
安装之前配置的代码把静态文件上传到git。
hexo g -d
对主页做完修改，我喜欢连着用。
hexo s --debug
这个指令用于启用本地localhost调试，地址：http://localhost:4000，适用于本地调试网页，ps，为了避免之前的cache，建议适用指令清空
hexo clean
最后CNAME解析到自己购买的域名。
博主穷，只能在万网买了这个一块钱的网址（哭）。
